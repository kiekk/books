## 2장 JUnit 핵심 들여다보기

```markdown
테스트 클래스가 되는 조건은 두 가지이다.
~~일단 public 클래스여야 하고, 파라미터를 받지 않는 생성자를 제공해해야 한다.~~
-> JUnit5 부터는 package-private(접근제한자 생략)이어도 테스트 클래스가 정상적으로 동작한다.
-> JUnit Jupiter는 리플렉션으로 객체를 만들기 때문에, 따로 생성자를 명시하지 않아도 컴파일러가 생성하는 기본 생성자를 사용한다.

...

모든 테스트 메서드는 각기 다른 테스트 클래스 인스턴스에서 실행되므로, 인스턴스 변수는 공유될 수 없다.
```

### 정리

```markdown
테스트 스위트는 관련된 테스트 클래스들을 묶어 그룹 단위로 실행하고자 할 때 사용한다.
스위트들을 묶어 더 큰 스위트를 만든느 것도 가능하다.
```
